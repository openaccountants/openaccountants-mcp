## When you DO NOT need a commercial license

You **do not** need a commercial license if you are:

- An individual preparing your own tax return using the skills in Claude, ChatGPT, or any other LLM
- An accountant or tax professional using the skills to assist with client work (the skills are a reference tool — your accounting software is not a derivative work)
- A contributor submitting improvements back to the Open Accountants repository
- A university, school, or educator using the skills for teaching
- Building an open-source project licensed under AGPL-3.0 (or a compatible copyleft license) that incorporates these skills
- A journalist, researcher, or blogger referencing, quoting, or reviewing the skills

You **do** need a commercial license if you are:

- Building a SaaS product, API, or platform that incorporates Open Accountants skills (or modified versions) into its functionality and is not released under AGPL-3.0

- Distributing the skills (or derivatives) as part of a proprietary product
- Using the skills in a commercial product where releasing your source code under AGPL-3.0 is not feasible

## Questions

**"I'm a solo accountant using the skills to help classify transactions for my clients. Do I need a license?"**
No. You are using the skills as a reference tool. Your accounting practice and software are not derivative works.

**"I'm building an AI accounting product that loads these skills as prompts. Do I need a license?"**
Yes, unless your product is also released under AGPL-3.0.

**"I want to fork the repo and add skills for my country. Do I need a license?"**
Not if your fork is also AGPL-3.0. If you want to keep your fork proprietary, yes.

**"I'm using one skill in a blog post or training course."**
No license needed. Attribution appreciated.
